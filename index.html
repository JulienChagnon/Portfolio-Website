Running and Jumping Detection using Machine Learning - Python

📄 Final Report

</>

See Code

As part of a Data Science group project, our 3-student team developed a complete human activity recognition system that processes raw smartphone accelerometer data to distinguish between walking and jumping. Following a structured workflow encompassing data collection, hierarchical HDF5 storage, signal visualization, preprocessing, feature extraction and normalization, classifier training, and GUI deployment, we leveraged Python’s scientific stack (pandas, h5py, scipy, scikit‑learn) to import CSV files an accelerometer app, clean and smooth the time series with forward‑fill and moving‑average filters, extract a suite of 40 statistical features per 5‑second window, train a logistic regression classifier to over 95% accuracy, and finally package the process into a Tkinter desktop application.

My contributions encompassed model training and GUI for data visualization. I helped design and train the logistic regression classifier by preprocessing labeled accelerometer data, evaluating model performance through metrics like accuracy, precision, and recall, also analyzing confusion matrices to identify common misclassifications between walking and jumping. On the GUI side, I built the user interface using Tkinter and implemented the segment_and_extract function to dynamically partition input CSV data into time-based windows. I integrated the trained logistic regression model and normalization scaler (using joblib) into the classify_csv function, enabling real-time predictions on user data. Classification results were visualized using matplotlib, with colour coded overlays indicating detected activity types and a scrollable output displaying timestamped labels.

We used Git throughout the development process to collaborate on code and manage version control. The resulting GUI offered an intuitive, real-time visualization of classification output, meeting our goal of creating a self-contained, user-friendly activity-tracking tool. A full breakdown of the implementation and user interface can be seen in the demonstration video and final project report.


Fluid and Powder Dispensing Device - Arduino/C++

📄 Final Report

As part of a term‑long engineering design project in first year, I worked on the development of an Automated Fluid Dispenser designed for precise, autonomous mixing of pharmaceutical solutions. This interdisciplinary project involved mechanical design, electronics integration, and embedded systems programming to create a functional prototype within strict space, material, and safety constraints. The team prepared and submitted interim reports at key milestones including initial concept, detailed design review, and prototype validation.

My contributions to the project focused heavily on both the system design and the software development. I wrote the complete Arduino C++ codebase, which included coordinating three subsystems: a servo‑powered powder dispensing mechanism, a motor rotating turntable, and a peristaltic pump liquid delivery system. The code utilized arrays to store and iterate over preset dosing instructions for five test tubes dispensing between 1 g and 2 g of powder per tube and delivering 20 mL of liquid per tube, controlled with start and emergency stop buttons. I implemented precision timing and motion control logic to ensure accurate dosing, along with real‑time interrupt checks for safety overrides.

Beyond programming, I contributed to designing a full SolidWorks sketch of the final assembly for 3D‑printed components, ensuring the gearbox, limit switch system, and test tube platform met both functional and dimensional requirements. I also authored major portions of the final report, including the Final Design, References, and Arduino Code documentation, articulating the engineering process behind design decisions.
This project strengthened my skills in embedded systems, sensor integration, Arduino programming, and mechanical prototyping, while reinforcing my ability to communicate technical concepts clearly within a collaborative team environment. In real‑world terms, such an automated dispenser could be deployed in pharmaceutical laboratories to mix concentrated drug formulations such as naloxone hydrochloride (Narcan) solutions for clinical research entirely autonomously, reducing contamination risk and improving reproducibility.


911 Operator Training Device - Arduino/C++

As part of a client‑based design project, our team partnered with the Toronto Police Service to modernize the Perfex operator‑training device, a 911 telephone operator testing platform built around manual time checks and paper-based modules that is becoming outdated. We translated all five original test stations (short‑story recall, reading‑aloud, copying critical information, simulated telephone dispatch, and map indexing) into a cohesive web interface seamlessly integrated with a custom Arduino‑powered taskbox housed in a portable briefcase with the goal of increasing test accuracy and repeatability.

Specifically, my roles included translating the physical layout of the original Perfex device into the physical Arduino taskbox, mapping each push‑button, slider, and rotary encoder to mirror the look and feel of the legacy device’s five stations. I developed the embedded Arduino code that generated physical stimuli tests through user prompts, timestamping every user action down to the millisecond. This allowed me to implement a real‑time scoring system that automatically calculated user performance by comparing the accuracy of button presses, slider positions, or dial settings against target values and by measuring reaction times. These automated metrics enabled objective, repeatable assessments that closely aligned with the client goals of increasing test fairness and ease of administration.


Portfolio Website - HTML, CSS, JavaScript

As a personal web development project, I designed and built this portfolio website from the ground up using HTML, CSS, and JavaScript to showcase my skills, projects, and experience in an interactive format.

In the HTML, elements such as <header>, <nav>, <main>, <section>, and <footer> were used to outline the page hierarchy. One challenge was ensuring that text wrapped well around the left and right floating images. To do so, I pulled each <img> out of the nested <p> tags and wrapped them in dedicated <div class="content-float float-left"> or float-right containers which I defined in CSS.

In the CSS, I also created tactile feedback by tweaking hover states on all buttons, softening background shade and lifting buttons by one pixel when cursor is hovering above. Media queries under 768px also disable floats on narrow viewports such as smartphones, stacking content vertically and converting all images into full width for readability, ensuring that on smaller screens text remains uniform rather than wrapping beside images. To observe this firsthand, try changing your browser window width to watch the content wrap dynamically!

I developed JavaScript logic that powers all dynamic behaviors. To manage lengthy work history entries, I wrote a toggle function bound to the “See More” button that adds or removes a ".show" class on the hidden job list and updates the button text accordingly. For the sidebar animation, I captured the header’s height and then animated the sidebar’s top position on each requestAnimationFrame call, using (currentY += (targetY – currentY) * 0.4) to achieve a fluid follow effect that locks once past a scroll threshold. I also deferred script.js at the end of the HTML <body> so the DOM (Document Object Model) was parsed before any event binding, preventing undefined element errors.

I have numerous small scale Java projects, my most significant being a console‑based Java game in which players guess the birthdates of a variety of “entities” (historical figures, countries, singers, etc.). The project is organized under the “PersonAndDate” package with seven classes: Date.java for parsing and validating dates, an abstract Entity.java defining shared fields (name, born, difficulty) and common methods (scoring, messages, cloning), four subclasses (Person, Country, Singer, Politician) each providing entityType(), clone(), and custom toString(), and GuessMaster.java as the driver that manages up to 100 cloned entities, handles user I/O via Scanner, and selects challenges at random. Across classes, the project comprises over 650 lines of code.

Through development I employed key Java techniques such as inheritance and polymorphism to share and override behavior within the entity hierarchy. I applied defensive copying in both the Date and Entity constructors and accessors to preserve encapsulation and protect internal state. I implemented the Cloneable interface and provided explicit copy constructors to enable safe object duplication. I also overrode the equals() and toString() methods to support logical comparison and produce clear, readable output.
